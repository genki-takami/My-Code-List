# Psyなび for 文化祭&お祭り

## iOS版

### 説明
```
「Psyなび for 文化祭&お祭り」は文化祭や学園祭、地域の祭りの案内パンフレットをデジタル化し、
デジタル化された文化祭コンテンツをiPhoneで閲覧・投稿を可能にするユーザー作成投稿型アプリです。
```

### 技術
```
言語：Swift
IDE：Xcode
ツール：CocoaPods
外部ライブラリ：RealmSwift、Firebase、SVProgressHUD
```

### 開発ヒストリー
```
＜開発背景＞
所属していた柔道部が大学の文化祭に参加した際、実行委員会が導入していた文化祭アプリに当部活の模擬
店情報が欠けていた。修正を申請したが、開発会社が休業日であったのとAppleの申請には時間がかかるが
故に、模擬店情報が記載されることなく文化祭が終了。そこから、開発企業に依存しない文化祭実行委員会
がアプリ上の文化祭データをいつでも追加・編集・削除できるアプリが必要であると気づく。また、全国の
文化祭や学園祭のナビゲーションアプリを集積するプラットフォームが無かった事と、散在する個別の文化
祭アプリはその年次のもので完結し持続可能ではなく、学校によっては発注料金も高かった。そのため学生
のためのユーザー作成投稿型文化祭ナビゲーションアプリを開発することを決心。

＜初期＞
文化祭リストタブ・お気に入りリストタブ・お知らせリストタブ・作成などタブで構成されたナビゲーション
画面を起点に、ブラウジング機能・マイリスト機能・コメント機能・投稿編集機能を実装。アカウント認証
やデータストレージのバックエンドをFirebaseで構築。SDKを利用し、リアルタイムなデータ表示を試みる。
ブラウジングに関しては、表示した画面で逐次データを受信していたことでやや非効率であった。また、お
知らせの表示順序が更新順ではなかっただけでなく、文化祭コンテンツを削除する処理をすべてアプリ上で
おこなっていた。さらに、企画イベントに追加する一部の画像を削除するには企画イベントごと削除しなけれ
ばならなかった。

＜中期＞
ブラウジング機能に関して、特定の条件下において事前にリスナーを取得しておくことで瞬間的なデータ表
示が改善した。アプリ内にて複雑かつ時間がかかる処理(削除など)をGCPのCloud Functionで実装するよ
うに修正。自動でログアウトされる仕様から、自発的にログアウトできるように修正。App内課金で投票機能
を追加できるオプションを設置するなど、新たなUIや機能を追加していくことでアプリをよりリッチに。

＜後期＞
企画イベントに追加する一部の画像を削除可能に修正。お知らせデータに特殊なタイムスタンプを付属して、
更新順に表示されるように改善し、またデータをバッチ処理で更新することでデータ送信の不具合も改善。
クイックアクション機能の追加に挑戦したが苦戦中。
```
