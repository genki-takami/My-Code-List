# Psyなび C-mate

## iOS版(MVVMアーキテクチャにリファクタリング中)

### 説明
```
「Psyなび for 文化祭&お祭り」アプリで表示されるPsyなびコンテンツの、ショップ/展示に関するデータ
を個別に編集できるサポートアプリです。
※データIDが必要
```

### 技術
```
言語：Swift
IDE：Xcode
ツール：CocoaPods
外部ライブラリ：Firebase、SVProgressHUD
```

### 開発ヒストリー
```
＜開発背景とビジネスモデル＞
このアプリは「Psyなび」というサービスの一部を成すアプリです。
「Psyなび」はウェブ(Psyなび Studio)とスマートフォンアプリ(Psyなび for 文化祭&お祭りとこのアプリ)で提供されています。
「Psyなび」はコンテンツを作りたいユーザとそのコンテンツを見たいユーザがターゲットです。
「Psyなび for 文化祭&お祭り」や「Psyなび Studio」で作成されるコンテンツは学園祭などのイベントの情報であることが期待され、情報量が大きくなります。
しかし、コンテンツを作成する際に登録されるアカウントは一つであり、同時に中央集権的な側面を持ちます。
「Psyなび」はユーザがリアルタイムでコンテンツのデータを作成・更新できることを特徴としているため、新しいデータをアップロードするためのアプローチが複数ある必要があります。
そこでこのアプリは、コンテンツの一部のデータ(模擬店や展示)を他のユーザが更新できる特徴を持っています。
そのユーザがデータを更新するには、データ(模擬店や展示)のIDを上に記したアカウントから教えてもらう必要があります。
この行為はアプリ外のテキストチャットサービスなどで行われることが期待されます。このアプリが使用される具体例を以下に記します。
イベントを主催する組織がアカウントを登録してコンテンツを作成します。
この時点では、ほとんどのデータ(出店しているショップや企画イベントの情報)が完全な状態で挿入されるか、または簡易にデータが登録されるだけになります。
このイベントに出店参加しているチームや組織が、彼らの情報を後々から更新する際に、アカウントを管理する組織に更新を依頼するというのはやや効率が落ちます。
そこでアカウントを管理する組織から更新したいデータに紐ずくIDを委譲する手筈で取得し、このアプリ上でID認証をした上でデータにアクセスします。
こうすることで、分散アクセスを利用して情報更新のリアルタイム性を高めます。更新した内容は「Psyなび for 文化祭&お祭り」に反映されます。
このIDはアカウントを管理する組織がブラウザアプリ「Psyなび Studio」にサインインすることで確認できます。

＜機能拡充＞
App内課金によって動画追加オプションを追加可能にする。このオプションはこのアプリからしか追加できない。
追加された動画は「Psyなび for 文化祭&お祭り」アプリのショップ/展示データで視聴可能。
```
