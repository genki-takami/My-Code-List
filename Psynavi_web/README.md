# Psyなび Studio

## WEBアプリケーション

### 説明
```
このサイトは、ユーザーに文化祭や学園祭などの案内パンフレットをスマートフォンで表示提供するアプリ
「Psyなび for 文化祭&お祭り」にコンテンツを公開・編集するための編集ウェブアプリケーションです。
なお、「Psyなび for 文化祭&お祭り」に表示提供されるコンテンツは開発者自身が提供するのではなく、
適切な権利と責任を持つユーザー自身がで本アプリ内にてアカウント作成の上で作成、編集、削除すること
になる。
※iOS版アプリでのみ同等の機能が利用可能
```

### 技術
```
言語：HTML5、CSS3、JavaScript(Vue.js)
エディタ：Atom
ツール：npm、Vue CLI、Firebase CLI
ホスト：Firebase Hosting
外部ライブラリ１：vue-dropdown-menu、vue-k-tabs、firebase、vue-form-wizard
外部ライブラリ２：vue-js-modal、vue-rocker-switch、vue-router、vue-twitter-counter
外部ライブラリ３：vue-uuid、vue2-google-maps、vuedraggable、vuex
```

### 開発ヒストリー
```
＜開発背景＞
iOS版「Psyなび for 文化祭&お祭り」での編集作業は、画面が小さい上にデータの全体像が扱い難いとい
う課題があった。また、Android版ではコンテンツの閲覧機能のみに限定されていたことから、機種に関係
なくPsyなびコンテンツを編集するために、ウェブ上でコンソール的なアプリを提供する必要があった。

＜初期＞
Vue.jsでサイトをSPAにし、認証やデータストレージのバックエンドをFirebaseで構築。SDKとVuexを
用いてデータの送受信を管理・実装し、ログインやアカウント作成をすることでコンソール画面に触れるこ
とができた。編集できるデータはメインデータ・アイコン画像２種・模擬店/展示データ・企画イベント・
お知らせ・サイトリンクデータ・マップデータ。ダッシュボードには編集したデータの状況や公開ボタンな
どが設置された。また、トップ画面上のプロモーションはテキストのみ。

＜中期＞
コンソールにアカウント表示名とパスワードの変更、アカウントの削除、データのリセットといった
デンジャーゾーンを設置。アプリ内にて複雑かつ時間がかかる処理をGCPのCloud Functionで実装するよ
うに修正。ダッシュボードにスマートフォンアプリ上でどのように表示されるか確認するためのモバイル
ディスプレイ的プレビューを設置。また、誤ってコンテンツを公開するのを防ぐために、逐一確認をする
公開フローフォームを作成。さらに、マップデータに手製のデジタル案内マップを挿入できるように修正。

＜後期＞
投票コンテンツの作成及び投票状況の管理を行える投票機能を追加。様々なライブラリを用いてコンソール
画面のユーザビリティを向上。データの送受信を効率的にするために非同期処理を抜本的に修正し、コード
のダイエットにも貢献。さらに、テキストのみであったトップ画面上のプロモーションにコンソール画面画像
を設置し、デザインも刷新。また、これまで本番環境のみで開発していたこともあり、環境変数を使い分け
ることにより開発環境と本番環境を分けることにした。
```
